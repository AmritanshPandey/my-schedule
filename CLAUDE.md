# CLAUDE.md

## Design Context

This project has impeccable design context. Before design/UI work, read:

- **PRODUCT.md** — strategic context. Register: `product`. Users: driven individuals
  running a personal "execution OS" (mobile = daily execution, desktop = strategy).
  Outcome: consistent execution (visible follow-through, streaks, trends).
- **DESIGN.md** — visual system (colors, typography, components) once generated.

**The 5 design principles (from PRODUCT.md):**
1. Execution over planning theater
2. Make progress feel earned, not gamified
3. Two surfaces, one system (mobile execution / desktop strategy)
4. Energetic, but calm underneath
5. Local-first and instant

**Personality:** sharp, motivating, energetic — disciplined, never cute.
**Anti-references:** gamified/childish habit apps, cluttered enterprise PM tools, generic
AI-SaaS template, sterile corporate calendar.
**Accessibility:** WCAG 2.1 AA in both themes, honor `prefers-reduced-motion`, ≥44px tap
targets.
